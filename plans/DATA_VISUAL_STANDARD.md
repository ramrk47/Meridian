# Data & Visual Experience Standard — judgment, not inventory (STANDING RULE)

> Companion to `MOBILE_DESIGN_STANDARD.md`. That one governs *density*; this one governs *substance*
> (what data we show) and *depiction* (how we draw it). Meridian's reason to exist is the **curated
> cross-platform judgment** on top of the platforms — not counts the platforms already show themselves.
> Hard rule for all current/future UI. Measurable bars at the bottom; a surface ships only if it passes.

## The two laws
1. **Show judgment, not inventory.** MCQ/module counts are inventory — every platform shows its own.
   Meridian must surface *judgment*: consensus high-yield, best-platform/faculty per subject, what-to-do-next,
   neutral reliability. If a screen only shows "how much content exists," it has not earned its place.
2. **Draw relationships, not numbers.** Use visualization to show how subjects/platforms/topics *relate*
   (coverage, agreement, where the mass is, where you're weak) — not big single numerals in cards.

## Neutrality firewall (non-negotiable — this IS the product)
- **Never fabricate.** Every curated claim traces to a source already in `RESEARCH_FINDINGS.md` /
  `MARKET_INTEL.md` (or the user-provided captures). No inventing ratings, ranks, or counts.
- **Label the epistemic status of every figure:**
  - `measured` — counts we hold (MCQ/module totals from captures).
  - `proxy` — derived, not the real thing (our current "high-yield" = MCQ-share, NOT exam yield).
  - `directional` — community-reputation/sentiment (e.g. best-faculty-by-subject from blogs/Quora).
  - `public-3p` — third-party public data (app-store reliability ratings), with date + source.
- **Money never moves the score.** Any future sponsored surface is labeled and separate (per strategy).
- **Publish methodology.** A visible "How we rate / sources" surface; cite source + capture date inline.

## Curation track — fix the substance (data-first pass, 1c.1)
1. **Stop dressing proxy as yield.** The current hyScore is MCQ-share within a subject — relabel it
   honestly (e.g. "MCQ density / volume") wherever shown. Where real PYQ data exists, introduce a
   distinct **PYQ-weighted yield** and keep the two visibly separate. Never call a proxy "high-yield"
   unbadged.
2. **Best-platform/faculty per subject matrix** — structure the H3 findings (Marrow↔Surgery/ENT/Ophtho/
   Patho/OBG; PrepLadder↔Anatomy/Pharma/Medicine/Ortho/Derma) into curated data: per subject, reputed
   strong platform(s), **flagged `directional`**, with source + date. Framed as "community reputation,"
   never as Meridian's verdict.
3. **Neutral reliability scorecard** — the app-store table (Marrow 4.7/33k, PrepLadder 4.4/21k,
   DocTutorials 4.2/1.7k, Cerebellum 3.7/3.4k, eGurukul 3.0/2.3k + recurring 1–2★ themes), `public-3p`,
   dated, sourced. A data point no incumbent can publish about rivals.
4. **Schema, not hardcoding** — these become curated fields on `D` (e.g. `D.platforms[].reliability`,
   `D.subjectStrength[]`, a `D.sources[]` registry), generated by `build_data.py` from a curated source
   file. PrepLadder/eGurukul count-drilling is a *separate, optional* Chrome-session prerequisite — do
   NOT block 1c.1 on it; leave the seam.

## Depiction track — fix the visuals (viz pass, 1c.2; desktop-led, mobile-aware)
Desktop is still the original sparse editorial layout (frozen during the mobile pass) — it wastes the
width. Use the width for **multi-panel data-viz**:
- **Subject × platform heatmap** — coverage/yield grid; the cross-platform thesis made visual, at a glance.
- **Consensus view** — where ≥2 independent platforms agree a topic is high-yield, visually distinct
  from disagreement.
- **Best-for-subject matrix** — the curated strength matrix + reliability, neutral and sourced.
- **Mass/where-to-spend** — treemap or ranked bars of MCQ/yield mass by subject.
- **Progress** — sparklines / small-multiples, not giant single numerals.
- Keep the warm almanac identity (paper/ink, serif display, calm accents). Color encodes data
  meaningfully (sequential for yield, categorical for platform) — never decorative neon.

## Anti-patterns (auto-fail)
- A screen that shows only counts/inventory; a proxy labeled "high-yield" with no badge; any rank/rating
  with no source + date; decorative charts that encode nothing; fabricated or unlabeled `directional` data;
  desktop that just stretches mobile cards across the width.

## Verification bar (every surface)
1. **Curation:** every figure on screen carries its epistemic label (`measured`/`proxy`/`directional`/
   `public-3p`); every rating/rank shows source + date; a "How we rate" surface exists. No proxy reads as yield.
2. **Depiction (desktop ≥1024):** at least one true relational visualization is present (heatmap /
   consensus / matrix / treemap) — not just stat cards + tables; the layout uses the width (multi-panel).
3. **Honesty:** spot-check 3 claims against `RESEARCH_FINDINGS.md`/`MARKET_INTEL.md` — all trace to a source.
4. No neutrality violation (nothing presents Meridian as judge of faculty where data is only directional).
5. Density (mobile) + no-horizontal-scroll + console-clean bars from the other standards still pass.
